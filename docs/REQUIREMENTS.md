## 一、项目概述

### 1.1 项目背景

Workflow Engine 是一个可视化的工作流编排平台，用于自动化执行复杂的多步骤任务。通过拖拽式节点编辑器，用户可以快速构建、调试和执行工作流，并集成 OpenClaw AI 能力和飞书通知功能。

### 1.2 项目目标

1. **易用性**：提供直观的可视化界面，降低工作流构建门槛
2. **灵活性**：支持多种节点类型和复杂的控制流（条件、并行、循环）
3. **可扩展性**：模块化架构，便于扩展新的节点类型和集成
4. **可靠性**：完善的错误处理、重试机制和状态持久化

### 1.3 技术选型

| 层次 | 技术 | 说明 |
|-----|------|------|
| 后端 | Java 8 + Spring Boot 2.7 | 稳定、企业级 |
| 前端 | 原生 HTML/CSS/JavaScript | 无框架依赖、轻量 |
| 数据库 | SQLite | 嵌入式、无需额外部署 |
| AI集成 | OpenClaw SDK | 本地AI能力调用 |
| 通知 | 飞书 API | 企业级通知推送 |

## 二、功能需求

### 2.1 工作流管理

#### 2.1.1 工作流列表

| 功能 | 描述 | 优先级 |
|-----|------|--------|
| 列表展示 | 显示所有工作流，包含名称、节点数、更新时间 | P0 |
| 搜索过滤 | 按名称/ID搜索工作流 | P1 |
| 文件夹分组 | 按文件夹分组展示工作流 | P1 |
| 拖拽排序 | 拖拽调整工作流顺序 | P2 |

#### 2.1.2 工作流操作

| 功能 | 描述 | 优先级 |
|-----|------|--------|
| 创建工作流 | 新建空白工作流，自动生成开始/结束节点 | P0 |
| 编辑工作流 | 修改名称、描述、节点配置 | P0 |
| 删除工作流 | 删除工作流（需确认），同时删除相关执行记录 | P0 |
| 克隆工作流 | 复制工作流及所有节点 | P1 |
| 导入工作流 | 从YAML文件导入工作流定义 | P2 |
| 导出工作流 | 导出为YAML格式文件 | P2 |

#### 2.1.3 工作流版本

| 功能 | 描述 | 优先级 |
|-----|------|--------|
| 版本号管理 | 支持设置版本号（如1.0, 1.1） | P2 |
| 版本历史 | 保留历史版本，支持回滚 | P3 |

### 2.2 文件夹管理

| 功能 | 描述 | 优先级 |
|-----|------|--------|
| 创建文件夹 | 创建新文件夹，支持嵌套 | P1 |
| 重命名文件夹 | 修改文件夹名称 | P1 |
| 删除文件夹 | 删除空文件夹（有工作流时需先移动） | P1 |
| 移动工作流 | 拖拽工作流到文件夹 | P1 |
| 展开/折叠 | 文件夹可展开折叠 | P2 |

### 2.3 节点编辑器

#### 2.3.1 画布操作

| 功能 | 描述 | 优先级 |
|-----|------|--------|
| 添加节点 | 右键菜单或工具栏添加节点 | P0 |
| 删除节点 | 删除选中节点及相关连线 | P0 |
| 拖拽移动 | 拖拽节点调整位置 | P0 |
| 多选操作 | Ctrl/Shift 多选节点批量操作 | P2 |
| 缩放平移 | 鼠标滚轮缩放，拖拽画布平移 | P1 |
| 对齐网格 | 可选的网格对齐功能 | P2 |
| 撤销重做 | Ctrl+Z 撤销，Ctrl+Y 重做 | P2 |

#### 2.3.2 节点类型

| 类型 | 描述 | 必需字段 | 优先级 |
|-----|------|---------|--------|
| **开始节点** | 工作流入口，每个工作流仅一个 | id, name | P0 |
| **结束节点** | 工作流终点，标记执行完成 | id, name | P0 |
| **Agent执行** | 调用OpenClaw执行AI任务 | id, name, agent_id, prompt, timeout | P0 |
| **条件判断** | 根据条件选择分支 | id, name, condition, branches | P1 |
| **并行执行** | 同时执行多个分支 | id, name, branches[] | P1 |
| **循环执行** | 循环执行直到条件满足 | id, name, loop_condition, max_iterations | P1 |
| **人工审核** | 等待人工确认后继续 | id, name, approver, message | P2 |
| **API调用** | 调用外部API | id, name, url, method, headers, body | P2 |
| **变量设置** | 设置/修改变量值 | id, name, variables | P2 |
| **延迟等待** | 等待指定时间 | id, name, delay_seconds | P3 |

#### 2.3.3 节点属性

| 属性 | 描述 | 适用节点 |
|-----|------|---------|
| id | 唯一标识符 | 所有节点 |
| name | 显示名称 | 所有节点 |
| type | 节点类型 | 所有节点 |
| position_x | X坐标 | 所有节点 |
| position_y | Y坐标 | 所有节点 |
| prompt | 执行提示词 | Agent执行 |
| agent_id | Agent标识 | Agent执行 |
| timeout | 超时时间（秒） | Agent执行 |
| condition | 条件表达式 | 条件判断 |
| branches | 分支配置 | 并行、条件、循环 |
| on_success | 成功时跳转节点ID | 所有节点 |
| on_fail | 失败时跳转节点ID | 所有节点 |

#### 2.3.4 连线管理

| 功能 | 描述 | 优先级 |
|-----|------|--------|
| 创建连线 | 从节点端口拖拽到目标节点 | P0 |
| 删除连线 | 点击连线删除 | P0 |
| 成功分支 | 绿色连线，表示成功时执行 | P0 |
| 失败分支 | 红色连线，表示失败时执行 | P0 |
| 条件分支 | 不同条件显示不同颜色 | P1 |

### 2.4 执行控制

#### 2.4.1 执行操作

| 功能 | 描述 | 优先级 |
|-----|------|--------|
| 启动执行 | 从开始节点执行工作流 | P0 |
| 暂停执行 | 暂停当前执行，保留状态 | P0 |
| 继续执行 | 从暂停点继续执行 | P0 |
| 停止执行 | 终止执行，标记为已停止 | P0 |
| 重新执行 | 从头重新执行 | P1 |
| 重试节点 | 重试失败的节点 | P1 |

#### 2.4.2 执行参数

| 参数 | 描述 | 默认值 |
|-----|------|--------|
| maxRetries | 单节点最大重试次数 | 3 |
| maxGlobalRetries | 全局最大重试次数 | 10 |
| timeout | 单节点超时时间（秒） | 300 |
| enableFeishu | 是否启用飞书通知 | false |

#### 2.4.3 执行状态

| 状态 | 描述 | 可用操作 |
|-----|------|---------|
| pending | 等待执行 | 启动 |
| running | 执行中 | 暂停、停止 |
| paused | 已暂停 | 继续、停止 |
| completed | 执行完成 | 重新执行 |
| failed | 执行失败 | 重试节点、重新执行 |
| stopped | 已停止 | 重新执行 |

#### 2.5.1 历史记录列表

#### 2.5.2 执行详情
