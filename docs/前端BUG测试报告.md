## 一、弹窗显示问题

### BUG #1: 页面初始化时所有弹窗同时显示
- **严重程度**: 严重
- **问题描述**: 页面刷新或首次加载后，所有模态弹窗（新建工作流、AI生成、全局配置、任务配置、添加节点、选择模板、选择路径、执行历史）都处于显示状态，互相叠加。
- **相关元素**:
  - CSS类: `.modal`
  - 弹窗数量: 7个
  - 弹窗标题: 新建工作流、✨ AI 生成工作流、⚙️ 全局配置、📋 任务配置、添加节点、选择模板、📁 选择路径
- **预期行为**: 页面加载后所有弹窗应处于隐藏状态，只有点击对应按钮才显示。
- **实际行为**: 所有弹窗同时显示，需要手动关闭或JS强制隐藏。
- **解决方案**:
  - 检查CSS初始化，确保 `.modal { display: none; }` 为默认状态
  - 检查JS初始化代码，确认没有错误地设置 `display: flex/block`
  - 建议在HTML中直接设置 `style="display: none;"`

### BUG #2: ESC键无法关闭弹窗
- **严重程度**: 中等
- **问题描述**: 按ESC键无法关闭任何弹窗，需要多次尝试或使用JS强制关闭。
- **相关元素**:
  - CSS类: `.modal`
- **预期行为**: 按ESC键应关闭当前最上层的弹窗。
- **实际行为**: 按ESC键无响应或需要按很多次。
- **解决方案**:
  - 添加全局键盘事件监听器，监听 `keydown` 事件
  - 当检测到 `event.key === 'Escape'` 时关闭当前显示的弹窗

## 九、关键发现：弹窗显示根本原因

### BUG #21: 内层modal元素内联样式阻止弹窗显示
- **严重程度**: 严重
- **问题描述**: 所有弹窗的内层 `.modal` 元素有内联样式 `style="display: none;"`，导致即使外层 `.modal-overlay` 添加了 `.show` 类，弹窗仍然无法显示。
- **相关元素**:
  - CSS类: `.modal-overlay`, `.modal`
  - 弹窗ID: `createModal`, `aiGenerateModal`, `globalConfigModal`, `taskConfigModal`, `addNodeModal`, `templateModal`, `pathSelectModal`
- **技术分析**:
```javascript
// 弹窗状态检查结果 - 所有内层modal都有 display: none
[
  {"overlayId":"createModal","innerModalStyle":"display: none;"},
  {"overlayId":"aiGenerateModal","innerModalStyle":"width: 520px; display: none;"},
  {"overlayId":"globalConfigModal","innerModalStyle":"width: 640px; display: none;"},
  // ...
]
```
- **解决方案**:
  - 方案一：修改JS函数，显示弹窗时移除内层modal的display:none
  - 方案二：修改CSS，添加 `.modal-overlay.show .modal { display: block !important; }`
  - 方案三：移除HTML中所有内层modal的内联 `style="display: none;"`

### BUG #22: 工作流节点数量显示不更新
- **严重程度**: 低
- **问题描述**: 添加节点后，侧边栏工作流卡片的节点数量显示不更新。
- **相关元素**:
  - 工作流卡片显示: `3 个节点`
- **预期行为**: 添加节点后节点数量应更新为实际数量。
- **实际行为**: 添加节点后节点数量仍显示旧值。
- **解决方案**:
  - 添加节点成功后更新工作流卡片的节点数量显示

### 技术分析结果

经过深入调查，发现弹窗无法显示的**根本原因**：

**内层 `.modal` 元素有内联样式 `display: none;`**

```javascript
// 弹窗状态检查结果
[
  {"overlayId":"createModal","overlayHasShow":true,"innerModalStyle":"display: none;"},
  {"overlayId":"aiGenerateModal","overlayHasShow":false,"innerModalStyle":"width: 520px; display: none;"},
  {"overlayId":"globalConfigModal","overlayHasShow":false,"innerModalStyle":"width: 640px; display: none;"},
  {"overlayId":"taskConfigModal","overlayHasShow":false,"innerModalStyle":"width: 560px; display: none;"},
  {"overlayId":"addNodeModal","overlayHasShow":false,"innerModalStyle":"display: none;"},
  {"overlayId":"templateModal","overlayHasShow":false,"innerModalStyle":"width: 560px; display: none;"},
  {"overlayId":"pathSelectModal","overlayHasShow":false,"innerModalStyle":"width: 640px; display: none;"}
]
```

### 弹窗结构分析

```html
<!-- 弹窗结构 -->
<div class="modal-overlay" id="createModal">  <!-- 外层，通过 .show 类控制 -->
  <div class="modal" style="display: none;">   <!-- 内层，内联样式阻止显示 -->
    ...弹窗内容...
  </div>
</div>
```

### CSS 定义

```css
/* modal-overlay 默认状态 */
.modal-overlay {
  position: fixed;
  top: 0; left: 0; right: 0; bottom: 0;
  background: rgba(0,0,0,0.7);
  display: flex;
  align-items: center;
  justify-content: center;
  opacity: 0;
  pointer-events: none;
  z-index: 9999;
}

/* modal-overlay.show 类 */
.modal-overlay.show {
  opacity: 1;
  pointer-events: auto;
}
```

### 问题根因

1. **外层 `.modal-overlay` 的 `.show` 类** 只控制 `opacity` 和 `pointer-events`
2. **内层 `.modal` 元素** 有内联 `style="display: none;"`
3. **点击按钮时** JS 只给外层添加 `.show` 类，没有移除内层的 `display: none`
4. **结果**：外层显示但内层隐藏，弹窗不可见

### 解决方案

**方案一：修改 JS 函数**
```javascript
function showCreateModal() {
  document.getElementById('newWorkflowName').value = '';
  document.getElementById('newWorkflowDesc').value = '';
  const overlay = document.getElementById('createModal');
  overlay.classList.add('show');
  // 移除内层 modal 的 display: none
  overlay.querySelector('.modal').style.display = '';
}
```

**方案二：修改 CSS**
```css
/* 让 .show 类穿透到内层 modal */
.modal-overlay.show .modal {
  display: block !important;
}
```

**方案三：移除 HTML 中的内联样式**
在 `index.html` 中移除所有 `.modal` 元素的 `style="display: none;"`
