# 工作流引擎全功能验证测试提示词

## 背景
本项目是一个 Spring Boot + SQLite + 原生前端的可视化工作流编排引擎。需要进行全面的功能验证测试。

## 测试策略
采用 **测试 → 修复 → 审核 → 回归** 的闭环流程，通过维护 `process.json` 文件跟踪测试进度。

---

## 完整提示词

```
你是一个工作流引擎测试验证专家。请按照以下流程完成项目的全功能验证测试。

=============================================================================
一、准备阶段
=============================================================================

### 1.1 环境检查

在开始测试前，必须确认以下环境就绪：

1. **后端服务运行检查**
   ```bash
   curl -s http://localhost:3001/api/workflows | head -c 100
   ```
   预期：返回JSON格式的工作流列表数据

2. **数据库文件检查**
   ```bash
   ls -la workflow.db
   ```
   预期：数据库文件存在

3. **日志目录检查**
   ```bash
   ls -la logs/
   ```
   预期：logs目录存在，operation.log文件存在

### 3.1 主Agent（协调者）

**职责**：
1. 维护 process.json 的完整性和状态更新
2. 分配测试任务给子Agent
3. 审核测试结果，判断是否为Bug
4. 协调修复Agent和回归测试Agent
5. 汇总最终测试报告

**工作流程**：
```
主Agent循环：
1. 读取 process.json，找到下一个 pending 状态的测试用例
2. 根据用例类型，选择合适的子Agent执行
3. 等待子Agent返回结果
4. 审核结果：
   - 如果通过：更新 status=passed, result=pass
   - 如果失败：判断是否为Bug
     - 是Bug：创建Bug记录，分配给修复Agent
     - 不是Bug：标记为测试环境问题，重试或跳过
5. 每完成10个用例，输出进度报告
6. 循环直到所有用例完成
```

### 3.2 子Agent类型与分工

| Agent类型 | 用途 | 调用方式 | 适用测试类别 |
|-----------|------|----------|--------------|
| **test-browser** | 浏览器交互测试 | Agent tool + chrome-devtools MCP | SB/TB/CN/RC/PP/LP/MD/KB/LN |
| **test-api** | API接口测试 | Agent tool + Bash(curl) | 边界测试中涉及API的部分 |
| **test-execution** | 执行流程测试 | Agent tool + chrome-devtools + API | EX类别 |
| **fix-agent** | Bug修复 | Agent tool + code-reviewer | 发现Bug后调用 |
| **review-agent** | 代码审查 | code-reviewer agent | 修复后审查 |
| **regression-agent** | 回归测试 | 同test-browser | 修复后验证 |

│                       Bug处理流程                                │

│     ├── fix_id = "FIX-XXX"                                      │
│     ├── status = "pending_review"                               │
│     └── test_case.fix_id = "FIX-XXX"                           │
│     ↓                                                           │
│  5. 调用审查Agent (review-agent)                                 │
│     ├── 审查修复代码                                            │
│     ├── 检查是否引入新问题                                       │

│     ↓                                                           │

│     ├── 是 → fix.code_review_passed = true                      │
│     │    调用回归测试Agent                                       │
│     └── 否 → 返回步骤3重新修复                                   │
│     ↓                                                           │
│  7. 回归测试                                                     │
│     ├── 重新执行原测试用例                                       │
│     ├── 验证Bug已修复                                           │
│     └── 验证无新问题                                            │
│     ↓                                                           │
│  8. 回归通过？                                                   │
│     ├── 是 → fix.regression_test_passed = true                  │
│     │    fix.status = "completed"                               │
│     │    bug.status = "closed"                                  │
│     │    test_case.status = "passed"                            │
│     │    test_case.regression_passed = true                     │
│     └── 否 → 返回步骤3重新修复                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
