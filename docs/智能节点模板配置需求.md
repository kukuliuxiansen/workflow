# 智能节点模板配置页面 - 需求文档

### 一、背景与目标

智能节点在执行时需要根据模板进行任务拆分和执行决策。当前模板存储在数据库中，以纯文本形式维护，缺乏可视化的配置界面。

本需求旨在开发一个模板配置页面，让用户能够通过模块化的方式配置智能节点的决策、审核、重试三种模板。

---

### 二、核心概念

#### 2.1 模板结构

一个完整的模板包含三个部分：决策模板、审核模板、重试模板。

用户创建智能节点时，直接关联一个模板ID。

#### 2.2 模块化设计

每个模板由多个模块组成，模块支持：
- 启用/禁用
- 排序（拖拽）
- 内容编辑

#### 2.3 模块类型

| 类型 | 说明 |
|------|------|
| 固定模块 | 预设的模块，前端硬编码 |
| 自定义模块 | 用户自行添加的模块 |

---

### 三、功能需求

#### 3.1 模板列表页

**展示内容**：
- 模板名称
- 模板ID
- 是否默认模板
- 创建/更新时间

**操作**：
- 新建模板
- 编辑模板
- 复制模板
- 删除模板（默认模板不可删除）
- 设为默认模板

#### 3.2 模板编辑页

**基础信息**：
- 模板名称（必填）
- 模板ID（自动生成，不可编辑）

**模板内容编辑**：
- 通过Tab切换：决策模板 / 审核模板 / 重试模板
- 每种模板由模块列表组成
- 模块支持拖拽排序
- 模块支持启用/禁用切换
- 模块支持删除（必须模块不可删除）
- 模块内容支持 `{{变量}}` 语法高亮
- 编辑器支持 `@` 符号快速插入变量

**模块操作**：
- 添加预设模块（从预设模块列表中选择）
- 添加自定义模块

**变量提示**：
- 根据当前Tab（决策/审核/重试）展示对应的可用变量
- 变量列表从后端接口获取

#### 3.3 模板预览

**功能**：
- 填入测试数据
- 预览渲染后的完整模板内容

#### 3.4 模板保存

**校验**：
- 模板名称不能为空
- 必须模块不能被禁用或删除

**保存逻辑**：
- 保存时将模块列表渲染为完整文本内容
- 同时存储模块配置JSON，用于编辑时还原

### 四、模块定义

#### 4.1 决策模板模块

**必须模块（不可删除、不可禁用）**：

| 模块ID | 名称 | 内容概述 |
|--------|------|----------|
| role_setting | 角色设定 | 定义AI的角色和职责 |
| project_info | 项目信息 | 项目路径、技术栈 |
| current_task | 当前任务 | 任务描述 |
| completed_tasks | 已完成任务 | 已完成的前置任务列表 |
| split_principle | 拆分原则 | 目标与验收分离等原则 |
| desc_rule | 任务描述规则 | description字段的写法规则 |
| criteria_rule | 验收标准规则 | criteria字段的写法规则 |
| decision_rule | 执行与拆分规则 | 判断执行还是拆分的条件 |
| output_format | 输出格式 | JSON输出格式示例 |
| task_instruction | 任务指令 | 结束语，指示AI开始分析 |

**可选模块（预设，用户可选择添加）**：

| 模块ID | 名称 | 内容概述 |
|--------|------|----------|
| important_reminder | 重要提醒 | 端口冲突规避等提醒事项 |
| skill_selection | skill选择 | developer/tester/deployer选择规则 |
| bug_handling | BUG处理规则 | 发现BUG时创建修复任务的规则 |
| split_dimension | 拆分维度选择 | 功能/页面/模块维度说明 |

#### 4.2 审核模板模块

**必须模块**：

| 模块ID | 名称 | 内容概述 |
|--------|------|----------|
| role_setting | 角色设定 | 定义审核者的角色 |
| current_task | 当前任务 | 任务描述 |
| criteria | 验收标准 | 审核标准 |
| execution_result | 执行结果 | 需要审核的执行结果 |

**可选模块**：

| 模块ID | 名称 | 内容概述 |
|--------|------|----------|
| execution_thought | 执行思考 | 执行过程的分析 |
| previous_issues | 之前的问题 | 重试时的问题列表 |
| output_format | 输出格式 | 审核结果JSON格式 |

#### 4.3 重试模板模块

**必须模块**：

| 模块ID | 名称 | 内容概述 |
|--------|------|----------|
| role_setting | 角色设定 | 定义重试执行者的角色 |
| current_task | 当前任务 | 任务描述 |
| issue_list | 问题列表 | 需要解决的问题 |

**可选模块**：

| 模块ID | 名称 | 内容概述 |
|--------|------|----------|
| criteria | 验收标准 | 目标验收标准 |
| output_format | 输出格式 | 重试结果JSON格式 |

### 六、数据存储

#### 6.1 模板表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(50) | 模板ID，如 tpl_decision_dev |
| name | VARCHAR(100) | 模板名称 |
| decision_modules | TEXT | 决策模板模块JSON数组 |
| review_modules | TEXT | 审核模板模块JSON数组 |
| retry_modules | TEXT | 重试模板模块JSON数组 |
| is_default | BOOLEAN | 是否默认模板 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

#### 6.2 模块JSON结构

每个模块包含以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 模块唯一标识 |
| type | String | fixed（固定）或 custom（自定义） |
| title | String | 模块标题 |
| content | String | 模块内容 |
| order | Integer | 排序序号 |
| enabled | Boolean | 是否启用 |
| required | Boolean | 是否必须（必须模块不可删除禁用） |

### 七、接口设计

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/templates | GET | 模板列表 |
| /api/templates/{id} | GET | 模板详情 |
| /api/templates | POST | 创建模板 |
| /api/templates/{id} | PUT | 更新模板 |
| /api/templates/{id} | DELETE | 删除模板 |
| /api/templates/{id}/set-default | PUT | 设为默认模板 |
| /api/templates/variables | GET | 获取变量列表（按类型） |
| /api/templates/preview | POST | 预览模板渲染结果 |

### 八、注意事项

1. **必须模块保护**：必须模块在前端禁止删除和禁用操作，后端保存时也需校验。

2. **变量高亮**：编辑器中 `{{变量名}}` 需要高亮显示，颜色与普通文本区分。

3. **@快捷插入**：输入 `@` 时弹出变量选择列表，选择后插入对应的 `{{变量名}}`。

4. **模块排序**：拖拽排序后需实时更新order字段，保存时一并提交。

5. **默认模板**：每个模板类型只能有一个默认模板，设置新默认时需取消旧的默认状态。

6. **删除保护**：默认模板不可删除，需先取消默认或删除整个模板。

7. **变量列表来源**：变量列表从后端获取，前端不硬编码，便于后续扩展。

8. **预设模块内容**：预设模块的默认内容硬编码在前端，添加时带入编辑器，用户可修改。
